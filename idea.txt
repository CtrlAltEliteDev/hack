What to build

Build a web app called something like ForgeFlow:

“An AI engineering workflow system that turns an idea, ticket, or PRD into an execution-ready plan and starter implementation.”

Core promise

User gives:

product idea / PRD / ticket
preferred stack
deadline / constraints

App returns:

scoped MVP
architecture
milestones
DB schema
API routes
file/folder scaffold
starter code pack
README + demo script

That is strong because it is not just chat. It creates deliverables.

Why this is a good hackathon project

It fits Track 1 very well because it is clearly an agentic workflow, not basic RAG. It also maps nicely to AMD’s setup because AMD Developer Cloud is built around direct access to Instinct MI300X GPUs, and AMD’s official docs currently highlight ROCm + vLLM as a supported path for LLM inference on those GPUs.

Why:
LangGraph is designed for stateful, long-running workflows, with explicit state, nodes, edges, durability, and human-in-the-loop support. That fits your product much better than a loose chat loop.

Inference

Why:

The architecture
High-level system
Next.js Web App
   |
   v
FastAPI Backend
   |
   +--> LangGraph Orchestrator
   |       |
   |       +--> Intake Agent
   |       +--> PM Agent
   |       +--> Architect Agent
   |       +--> Scaffold Agent
   |       +--> Reviewer Agent
   |       +--> Delivery Agent
   |
   +--> Postgres / pgvector
   |
   +--> vLLM Server on AMD Developer Cloud
Agent roles

1. Intake Agent

parses user input
extracts product type, stack, timeline, features, constraints

2. PM Agent

defines MVP scope
creates milestones
decides what is in/out

3. Architect Agent

proposes system design
chooses services, routes, schema, modules

4. Scaffold Agent

generates folder structure
API contract
starter files
code stubs

5. Reviewer Agent

checks gaps
flags unrealistic scope
finds missing components / risks

6. Delivery Agent

packages outputs
writes README
creates demo script / build checklist
Why this architecture is good

Because you are not making “agents talking for no reason.”

You are making a pipeline with clear business outputs.

That matters a lot.

LangGraph is a good fit here because it models workflows as graphs with shared state, nodes, and edges, so your system can be deterministic where needed and agentic where useful.

Why this has an edge
1. It is outcome-first

Most people will build:

chatbot
RAG assistant
generic agent demo

You are building:

idea -> execution pack

That is much more tangible.

2. It is broader than Cursor-like tools

Cursor/Codex mostly live in the code-writing loop.
Your product helps with:

scope
architecture
milestones
build order
starter implementation
demo readiness

That is a different layer of value.

3. It feels real

Judges usually trust products that output:

plans
documents
code artifacts
structured workflow results

more than “look, agents are chatting.”

4. It matches your profile

You have full-stack experience and multi-agent experience.
So your edge is combining:

orchestration
useful product design
polished UI
believable outputs
5. AMD fit is visible

Using AMD Developer Cloud + ROCm + vLLM makes the compute story explicit instead of hidden. AMD’s official material emphasizes MI300X access, ROCm tooling, and preconfigured environments for inference and other AI workflows.
